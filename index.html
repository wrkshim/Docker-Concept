🚢 핵심 비유: 선박 컨테이너

내용물이 뭐든 상관없이 규격화된 컨테이너에 담아 옮기면, 어느 항구(서버)에서든 똑같이 내리고 쓸 수 있습니다. 내 코드를 설정, 라이브러리와 함께 '도커 컨테이너'에 담으면, 어디서나 똑같이 실행됩니다!

도커의 3가지 핵심 요소

📜 이미지 (Image)

'설계도' 또는 '레시피'

컨테이너를 만드는 데 필요한 모든 정보가 담긴 파일. 이 설계도로 똑같은 컨테이너를 계속 찍어낼 수 있습니다.

📦 컨테이너 (Container)

'살아있는 프로그램'

이미지(설계도)를 바탕으로 실제 메모리에서 실행되는 상태. 컴퓨터 안의 독립된 작은 컴퓨터입니다.

🏪 도커 허브 (Docker Hub)

'이미지 앱스토어'

💡 Tip: vscode와 같은 IDE에서 도커 Plugin(도커 관리 및 설정 코드 자동 완성)의 도움을 받아 Dockerfile이라는 설정 파일을 쉽게 완성할 수 있고, 이것을 이용하면 어렵지 않게 도커 이미지(설계도)로 빌드(변환), 실행 및 공유할 수 있다.

도커 이미지와 컨테이너의 관계

이미지는 '설계도', 컨테이너는 '실행된 모습'입니다.

도커 이용 웹서버 구성하기

1단계: 웹서버 실행

아파치 웹서버 컨테이너를 'my-web' 이름으로 실행합니다.

docker run -d --name my-web -p 8080:80 httpd

이제 브라우저에서 localhost:8080/index.html으로 접속해 "It works!"를 확인하세요.

2단계: 컨테이너 접속

'my-web' 컨테이너의 터미널(bash)로 접속합니다. (-it 옵션으로 상호작용 가능하게!)

docker exec -it my-web bash

이제 여러분의 터미널은 컨테이너 '안'입니다!

3단계: 파일 수정

웹페이지 내용을 "Hello Docker!"로 변경하는 간단한 예입니다.( 2단계에서 컨테이너에 접속한 터미널에서 아래 명령을 실행합니다.)

echo "Hello Docker!" > /usr/local/apache2/htdocs/index.html

브라우저를 새로고침해서 변경된 내용을 확인하고, exit으로 빠져나오세요.

'나만의 이미지'는 어떻게 만드나요? 📝

Dockerfile: 나만의 이미지를 만드는 레시피 북

Dockerfile은 이미지를 만들기 위한 명령어들의 목록입니다. 이 파일을 만들어 두면, 언제 어디서든 똑같은 환경의 이미지를 생성할 수 있습니다.

1단계: 간단한 웹서버 Dockerfile 예시

# 1. Nginx 웹서버 이미지를 기반으로 시작합니다.
FROM nginx:alpine

# 2. 현재 폴더의 모든 발표 자료 파일들을 웹서버의 기본 폴더로 복사합니다.
COPY . /usr/share/nginx/html

# 3. 80번 포트를 외부에 노출합니다.
EXPOSE 80

이 세 줄만으로, 발표 자료 전체를 포함하는 웹서버 이미지가 완성됩니다!

2단계: 도커 이미지 빌드

터미널을 오픈하여 아래 명령어를 실행하면 my-presentation이라는 이름의 도커 이미지를 생성합니다.

docker build -t my-presentation .

3단계: 도커 이미지 실행

아래 명령어로 방금 만든 이미지를 컨테이너로 실행합니다.

docker run -d --name presentation-server -p 8080:80 my-presentation

Docker Compose: 명령어 하나로 전체 앱 실행 🚀

무엇인가요?

웹 서버, 데이터베이스처럼 여러 컨테이너를 '하나의 서비스'로 묶어, 명령어 하나로 동시에 실행하고 관리하게 해주는 도구입니다.

docker-compose.yml 파일을 사용하여 서비스, 네트워크, 볼륨을 간단히 정의할 수 있습니다.

주요 명령어

# 모든 서비스 시작 (백그라운드)
docker-compose up -d

# 모든 서비스 중지 및 컨테이너 삭제
docker-compose down

예시: docker-compose.yml

services:
  web:
    build: .
    command: python manage.py runserver 0.0.0.0:8000
    volumes:
      - .:/code
    ports:
      - "8000:8000"
    depends_on:
      - db
    environment:
      - MYSQL_DATABASE=mydb
      - MYSQL_USER=user
      - MYSQL_PASSWORD=password
      - MYSQL_HOST=db

  db:
    image: mysql:8.0
    environment:
      MYSQL_DATABASE: mydb
      MYSQL_USER: user
      MYSQL_PASSWORD: password
      MYSQL_ROOT_PASSWORD: rootpassword
    volumes:
      - db_data:/var/lib/mysql

volumes:
  db_data:

쿠버네티스 아키텍처 🏗️

마스터 노드가 워커 노드들을 지휘하는 구조입니다.

1. 🧠 마스터 노드 (Control Plane) - 관제탑

API 서버 (kube-api-server)

'총괄 매니저' (API 서버를 통해서만 명령/상태 관리)

etcd (et-see-dee)

'원하는 상태(Desired State)'와 '실제 상태(Current State)'를 모두 기록 관리 '데이터베이스'

스케줄러 (kube-scheduler)

역할: '배차 담당자'

컨트롤러 매니저 (kube-controller-manager)

etcd의 '원하는 상태'와 '실제 상태'를 끊임없이 비교하는 역할

2. 🏃 워커 노드 (Worker Nodes) - 부두

파드 (Pod)

컨테이너를 담는 '표준 규격 상자'

Kubelet (큐블릿)

실제로 컨테이너를 실행하고 관리

Kube-proxy (큐브-프록시)

네트워크 경로(길)를 만들고 관리

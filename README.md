방법 1: 도커를 이용하는 방법 (추천! 발표 내용과 일치)
----------------------------------------------------

발표 자료의 주제가 '도커'인 만큼, 발표 자료 자체를 도커 이미지로 만들어 
웹 서버로 실행하는 방법입니다. 이 방법을 사용하면 다른 컴퓨터에 
도커만 설치되어 있다면, 명령어 하나로 발표 자료를 바로 띄울 수 있습니다.

  1. Dockerfile 생성 (발표 자료가 있는 압축 파일에 이미 만들어져 있습니다.):
     ----------------
     발표 자료가 있는 폴더에 Dockerfile이라는 이름의 파일을 만들고 
     아래 내용을 추가합니다. ( 만약 내용이 없는 경우 혹은 사용자가 쉽게 텍스트 에디터로
     도커 이미지 생성을 위한 설계도인 Dockerfile을 쉽게 만들 수 있다는 것을 설명하기 위한 예시임 )

     ~~~ Dockerfile 파일 내용 ~~~
     -----------------------------------
     # 1. Nginx 웹서버 이미지를 기반으로 시작합니다.
     FROM nginx:alpine
     
     # 2. 현재 폴더의 모든 발표 자료 파일들을 웹서버의 기본 폴더로 복사합니다.
     COPY . /usr/share/nginx/html
     
     # 3. 80번 포트를 외부에 노출합니다.
     EXPOSE 80
     -----------------------------------

  2. 도커 이미지 빌드:
     ----------------
     터미널에서 아래 명령어를 실행하여 my-presentation
     이라는 이름의 도커 이미지를 만듭니다.
     
     $docker build -t my-presentation .

  3. 도커 이미지 실행:
     ----------------
     아래 명령어로 방금 만든 이미지를 컨테이너로 실행합니다.
     
     $docker run -d --name presentation-server -p 8080:80 my-presentation

  4. 확인:
     -----
     웹 브라우저에서 접속:
     http://localhost:8080/Docker%20Concept.html

  5. 정리 (종료 및 삭제):
     --------------------
     # 컨테이너 중지
     $docker stop presentation-server
     
     # 컨테이너 삭제
     $docker rm presentation-server
     
     # 이미지 삭제
     $docker rmi my-presentation

  6. 다른 컴퓨터로 이미지 옮기기:
     ---------------------------
     # 이미지를 파일로 저장
     $docker save -o docker-presentation.tar my-presentation
     
     # 다른 컴퓨터에서 이미지 로드
     $docker load -i docker-presentation.tar
     
     # 실행 (3번 단계와 동일)
     $docker run -d --name presentation-server -p 8080:80 my-presentation

방법 2: 웹 호스팅 서비스에 배포 (온라인 공유)
----------------------------------------------------

A. GitHub Pages (무료)
   --------------------
   1. GitHub 저장소 생성
   2. Docker Concept.html 및 이미지 파일들 업로드
   3. Settings > Pages에서 배포 설정
   4. https://사용자명.github.io/저장소명/Docker%20Concept.html 로 접속

B. Netlify (무료)
   --------------
   1. https://www.netlify.com 가입
   2. 드래그 앤 드롭으로 파일 업로드
   3. 자동으로 배포 URL 생성됨

C. Vercel (무료)
   -------------
   1. https://vercel.com 가입
   2. vercel CLI 설치: npm i -g vercel
   3. 프로젝트 폴더에서: vercel
   4. 자동으로 배포 URL 생성됨

D. AWS S3 + CloudFront
   --------------------
   1. S3 버킷 생성 및 정적 웹사이트 호스팅 활성화
   2. 파일들 업로드
   3. CloudFront 배포 생성 (선택사항, CDN 사용)
